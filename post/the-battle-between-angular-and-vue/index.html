The Battle Between Angular and Vue

August 26, 2018
frontend framework

Personally, I don’t mind to switch frameworks because each one has its pros and cons, and I always like to try new techs and learn from it. I’ve used jQuery, Backbone, Angular1, Vue.js so far, and really you can see how frontend evolve so fast and those frameworks and concepts make frontend development much easier than before.

Angular is a great framework and introduced two day data binding, dependency injection, modularization etc. And it was the hot choice after Backbone because of those, so lot’s of developers adopt it and made many successful projects. Vue, on the other hand, is later borned, and not many developers look at it in the first place.

I still remember when me with my teammates sit together discussing which framework to choose between Angular/React/Vue. At that time, Vue was not really on the table. I did some googling, then wrote a simple todo app with each Angular and React(not Vue). I made my point of why I dislike React way of doing things. Anyway, the conclusion after the meeting is Angular wins the battle.

Although we were all happy with finally we made the decision, I start wondering why Vue is constantly at the top page of github trending project and more and more articles and blogs people are writing about it. I start look at Vue documentation and I really couldn’t stop reading it because it is so clean and detailed, and it shows how easy it is to use. After two hours of reading the whole doc(right, I did that), I changed my mind. Yes, I choose Vue over Angular just simply because of that. After, I told my teammates Vue is a great framework and I’m glad they are all supportive.

Okay, now I have a chance to choose again, how do I choose now? Well, I haven’t really use Angular 6 in production so I couldn’t really say much. And I think they both are great framework to work with. I’d like to build with both if possible. However, I’d say Vue is more suitable with our current situation. It is more flexible and easy learning curve. It is rather a library than a framework. Considering our UI is not so complex, I feel Angular is much an overkill.

Saying that, there is a deal breaker for Angular. That is Typescript. If you want to use Angular, you have to use Typescript. Well, it is a bundled sell. Typescript is created a while back, at that time, Javascript has not much features. Ever since ES6, Javascript become much more powerful. It’s not that promising to use Typescript anymore. Here I don’t want to criticize Typescript and apparently a lot of people loves it, including the creator of Node. Personally I just don’t like build things on top of language that is not supported natively, it’s not only additional learning, but adding more layers will make things more complicated and bring more problems later on.

In the end, let’s take a quick look at github. Angular and Vue. Comparing the stars and forks, I’d say the trend is towards Vue.
